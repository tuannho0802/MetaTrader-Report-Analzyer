# Internationalization (i18n) System

The MetaTrader Report Analyzer features a custom-built, persistent internationalization system that supports English and Vietnamese.

## 🏗️ Architecture

The i18n system consists of three main parts:

### 1. Persistent State (`useSettingsStore`)
Located in `lib/store/useSettingsStore.ts`, this Zustand store manages the user's language preference. It uses the `persist` middleware to ensure the setting survives page refreshes.

```typescript
export type Language = 'en' | 'vi';
// ...
language: 'en',
setLanguage: (lang: Language) => set({ language: lang }),
```

### 2. Translation Dictionary (`lib/i18n.tsx`)
A nested object structure containing all UI strings. Keys are organized by component or domain (e.g., `common`, `dashboard`, `analysis`, `filter`).

### 3. Hooks and Components
- `useTranslation()`: The primary hook for retrieving translated strings.
  ```tsx
  const { t } = useTranslation();
  return <h1>{t('dashboard.ready')}</h1>;
  ```
- `Trans`: A simple component for translating static keys.
  ```tsx
  <Trans i18nKey="common.save" />
  ```

## ➕ Adding New Translations

1.  Open `lib/i18n.tsx`.
2.  Add your key to both the `en` and `vi` sections of the `translations` object.
3.  Ensure the paths are identical.

## 🔄 Language Persistence

Language state is synchronized across the application via the `TranslationProvider` in `app/layout.tsx`. When a user changes the language in the `Header.tsx` settings, the entire UI updates reactively.

## 🛠️ Developer Guidelines

- **No Hard-coded Strings**: Never include user-facing text directly in components.
- **Namespacing**: Group related keys under a logical parent (e.g., `filter.errors.patternRequired`).
- **Arrays**: For list-based content (like steps in a guide), store them as arrays in the dictionary.
