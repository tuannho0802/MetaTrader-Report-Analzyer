# Internationalization (i18n) System

The MetaTrader Report Analyzer features a custom-built, persistent internationalization system that supports English and Vietnamese.

The i18n system is designed for high performance and zero-flicker rendering in a static export environment.

### 1. Persistent Settings Store (`useSettingsStore`)
Located in `lib/store/useSettingsStore.ts`, this Zustand store manages the user's language preference (`en` | `vi`). 
- **Persistence**: Uses `persist` middleware to save settings to `localStorage`.
- **Initialization**: Hydrates automatically on app load, ensuring the user's preferred language is applied immediately.

### 2. Nested Translation Dictionary (`lib/i18n.tsx`)
UI strings are organized into a nested object structure. This allows for logical grouping and prevents key collisions in a large application.
- **Namespaces**:
  - `common`: Global terms (Save, Cancel, Dashboard, Light/Dark/System).
  - `settings`: Application configuration and Data Management.
  - `history`: Session management, badges, and confirmation dialogs.
  - `performance`: Root namespace for analytics (Filters, Monthly Returns, EA analysis).
  - `dashboard`: Landing page and general analysis strings.
  - `comparison`: EA Comparator metrics and visualization labels.
  - `filter`: Labels and error messages for the filtering form.
  - `explore`: Trade Explorer dashboard strings (hourly/daily/monthly analysis).
  - `statistics`: Global statistics and leaderboard strings.
  - `montecarlo`: Simulation-specific labels and results.

### 3. Translation Engine
The system uses a custom `useTranslation` hook:
- **Recursive Lookup**: Resolves dot-notation paths like `t('comparison.columns.sharpeRatio')`.
- **Template Support**: Supports basic string replacement (e.g., `From {start} to {end}`).

```tsx
// Usage Example
const { t } = useTranslation();

return (
  <CardTitle>{t('comparison.title')}</CardTitle>
  <p>{t('filter.errors.patternRequired')}</p>
);
```

### 4. Translation Provider
The `TranslationProvider` (integrated into `components/layout/ClientLayout.tsx`) wraps the application. It listens to the `useSettingsStore` and provides the current translation context to all child components.

## âž• Adding New Translations

1. **Register Key**: Add the new key path to both the `en` and `vi` objects in `lib/i18n.tsx`.
2. **Keep Parity**: Ensure that every key added to English has a corresponding Vietnamese translation to avoid "missing key" errors.
3. **Use Hierarchy**: Group related keys under the appropriate namespace.

## đź› ď¸Ź Developer Best Practices

- **Zero Hard-coding**: All user-facing strings MUST be moved to `lib/i18n.tsx`.
- **Contextual Naming**: Use keys that describe the location or purpose (e.g., `dashboard.uploadHint` instead of `dashboard.text1`).
- **Persistence Check**: After adding a key, verify that switching languages and refreshing the page correctly restores the user's choice.
